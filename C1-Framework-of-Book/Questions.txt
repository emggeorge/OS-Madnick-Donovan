1-1 What is the difference between the following?
    1 process and processor
    2 user, job, process

A1:
    process: "A process (or task) is a computation that may be done concurrently with other computations." (pg 5)
    processor: "A processor is a hardware device capable of inerpreting instructions and performing the indicated operation (e.g. addition)." (pg 4)
    difference: A process is a verb and a processor is a noun. The processor is a physical piece of hardware, while the process is an action that is made. These two are related by the fact that a processor is the object that executes a process.

A2:
    user: The thing creating the task at hand. It decides what needs to be done and how they want it done. "A user submits his job to the operating system. A user is anybody that desires work to be done by a computer system." (pg 5)
    job: The overarching thing that the user wants to create. "A job is the collection of activities needed to do the work required. A job may be divided into several steps (job steps)." (pg 5)
    process: An individual step towards performing the job. "A process (or task) is a computation that may be done concurrently with other computations." (pg 5)
    difference: The difference between these three things is how they are related. A user is the creator, the job is their creation, and a process is how they create it by way of individual computations.

1-3 One may make the following analogy:
    Boss is comparable to CPU
    Secretary is comparable to I/O channel
    Typewriter is comparable to I/O device

    1 State how a boss, secretary, and typwriter communicate with each other
      (e.g., boss with his secretary).
    2 How is each scheduled? Consider priorities, preemption, and interruption and which can be interupted by whom (e.g., who can interrupt the boss?).
    3 Repeat 1 and 2 for the group: CPU, I/O channel, I/O device

A1:
    boss + secretary: boss verbally speaks with his secretary, where the boss is dictating directions or the secretary is sharing information on the day that she has managed for him
    secretary + typewriter: the secretary uses her fingers to press on the typewriter and the typewriter responds by producing a page with the information she set out to make
    boss + typewriter: boss never interacts with the typewriter directly, but recieves its output.

A2:
    (forgive my misogyny in the following)
    A boss will dictate his priorities to the secretary who will then create her own priorities that enable her to efficiently use the typewriter to finish her tasks. The typewriter will never dictate a priority other than in a malfunction, and then is dictating a priority in a way because it is not responding as expected. Sometimes, if a secretary has information that the boss does not, she will provide input.

    A secretary can preempt the boss's tasks by completing things she know he likes finished often before he asks. A typwriter preemts the secretary's needs when is automatically dings at the end of a page to notify finishing (I think?).

    A typewriter will never interrupt a secretary except in cases of malfunction. A boss may interrupt the secretary if there is something he needs before she is finished typing. A secretary may interrupt the boss if there is an issue with the task he has asked for.

A3:
    CPU + I/O channel: "the CPU may execute a START I/O (SIO) instruction to initiate an I/O processor." (pg 4) The CPU communicates with the I/O channel through start up tasks or end tasks.
    I/O channel + I/O device: "the I/O (Input and Output) processors are designed to control I/O devices and handle the movement of data between the memory and the I/O devices." (pg 4). The I/O channel is the one that dictates to the I/O device what to do.
    CPU + I/O device: These do not communicate directly except in the case where an I/O channel would be lacking or an I/O device has sent a response through the I/O channel back to the CPU.

    A CPU dictates priorities to the I/O channel which then dictates to the I/O device its priorities to finish the task. Both the I/O channel and the I/O device may preempt tasks that the one above it may ask for. Again moving up the chain is how error interruptions would be communicated, and down the chain would be how stop commands are communicated.

1-5 Assume that you are head of the check-sorting division of a bank, which processes 50,000 checks a day for its 99,999 customers. That is, 50,000 checks are distributed among 99,999 bins.

Two sorting methods are proposed.
    1 Linear -- the bins are lined up in a row. A person takes about 99,999/10 seconds (about 1000 seconds) to find the right bin and deposit the check in it.
    2 Bucket sort -- 11,110 intermediary bins are used. Based on first digit of check, he places it in one of the ten bins. After all the checks have been so processed, he goes to one of these bins and processes each check into other bins based upon the second digit, etc. Processing such check action takes about 10/10 = 1 second.

    a How long would it take to process all the checks using methods 1 and 2?
    b Using method 2, which steps could be done in parallel (as separate processes)? That is, how many people could be used?
    c If parallel processing were used in method 2, how long would it take to sort all the checks?

Aa:
    Linear sort: 50,000 checks * 1000s/check = 50,000,000 seconds to process all the checks, or ~580 days
    Bucket sort: 50,000 checks * 1s/check = 50,000 seconds to process all the checks, or ~0.6 days

Ab:
    A person can stand at each group of sub-bins so that a person only needs to check one digit and place it in the correct bin.

    In this case, there are 11,110 bins. 10 bins for the first set of digits, 10 bins for each digit of the previous set, so 100 bins, 1000 bins in the next level (10 bins for each of the 100 bins) and 10000 bins at the final, adding up to 11,110 bins.

    If there is a person at each set of bins, the first level would have 1 person for the set of 10 bins, the next level would have 10 people for each sub bin, the third would have 100, and the fourth would have 1000.

    This comes out to mean there are 1111 people who can be working in parallel to sort the checks.

Ac:
    If sorting one check through all 4 layers of sub bins takes 1 second, then it can be assumed that each layer of sorting takes 0.25 seconds. Sorting in parallel means that the limits on time are only how long it takes the first person to process all the check, and then how long it takes that final check to be fully processed to the end.

    It would take the first person 50,000 checks * 0.25 s/check = 12,500 seconds
    The final check takes one second to fully process, so we add the final 0.75s to send it through the 3 remaining bins the total.
    That makes 12,500.75 seconds to process all the checks, or ~3.5 hours.

    This does assume that every check can be fully parallelized, which may not be 100% accurate if two checks need to go to the same sub-bin in less than the amount of time that it takes to process the previous check. I think that the difference would be negligible, though.

    1-6 We make an analogy between a warehouse with many shelves and a memory with many locations. Assume a large number of items have arrived and will continue to arrive. The task of the warehouse manager (memory manager) is to store these items on the shelves.
        Give an example of how the four memory manager functions of Section 1-3 are performed; for example,
        1 How does the manager keep track of space?
        2 If the loading dock has many items on it, how does the manager decide which item to store first?
        3 How are shelves allocated?
        4 What does the manager do when one item is removed from the shelves?

    A:
        Memory manager: A memory manager keeps track of space by keeping a log of what is in memory and how much is left. When an item arrives for its shelves, it checks if there is enough space for the item and places it in the most logical place for it.

        Processor manager: A processor manager must know how many resources each job will take, both time and memory, and never start a process without ensuring there will be enough of the resources. In this analogy, it will monitor how long it would take to unload a truck or organize a shelf, and only book these tasks when it is possible to do so.

        Device manager: A device manager would be like an equipment manager in the warehouse. It would keep track of things like forklifts, safety equipment, and loading docks. It converts an item into a form ready for the warehouse and decides who must take each item.

        Information manager: An information manager would know everything about the inventory. It decides if something can be removed from the shelves or if it can go back.

    1-7 We can make an analogy between managing a restaurant and managing processors when the following are analogous.
        Cook--processor
        Maitre d' (cook's boss)--processor manager
        Customers' orders--processes--they are all demanding processing, e.g. they want their meal.
        Customers--users
        Give an example of how the four processor manager functions of Section 1-4 are performed, e.g., how does the boss keep track of resources--the processors and processes, etc.?

    A:
        I'm having trouble finding four specific processor management functions in Section 1-4, so I'm going to use the functions in section 1-3.2 to answer this question and then use the framing of 1-4.

        Traffic controller: The boss accepts meal tickets and formats them so that a cook can make them. For example, a customer could want a hamburger, so the boss will determine what kind of patties need to be used and how long they need to be cooked.

        Process scheduler: This keeps track of how long the process will take, signals when to start, and stops the process if it is taking too long. This is similar to a boss training cooks to a certain recip.

        Dispatcher: The boss goes out to set up each work station so there are enough cooking utensils at any time a cook needs. He would also designate who is working on what, whether cutting, grilling, etc.

        Reclaimer: When the customers are done eating, the boss signals clean up and then sends everyone home.
